## рҹҢҢ What is Rippler?

Rippler is an **intelligent static analysis tool** that tears open your Spring Boot codebase and renders everything inside вҖ” every service, every class, every Kafka stream, every database вҖ” as a stunning, interactive, neon-lit architecture graph.

No runtime agents. No bytecode manipulation. No config changes to your project.

Just point Rippler at your code and watch your architecture come alive.

<br/>

**The problem it solves:**

> You join a new team. There are 12 microservices. The architecture diagram on Confluence is 2 years out of date. No one knows which service calls which, or what happens when `payment-service` goes down.
>
> **Rippler solves this in 30 seconds.**

<br/>

## вңЁ Features

<br/>

### рҹ”¬ Static Analysis Engine
Powered by **JavaParser AST** вҖ” Rippler reads your source code at the syntax tree level, detecting every annotation, injection, Feign client, Kafka listener, and JPA entity without ever running your application.

### рҹ—әпёҸ Dual View Architecture Graph

| View | Description |
|---|---|
| **Service Topology** | High-level map of how your microservices interconnect вҖ” perfect for architecture reviews |
| **Class Code Map** | Drill into any service's internal structure вҖ” controllers, services, repositories, and their wiring |

### вҡЎ Dual Layout Physics

| Layout | Best For |
|---|---|
| **Semantic Flow** (Dagre DAG) | Hierarchical codebases вҖ” see the layer structure clearly |
| **Force Orbit** (ELK.js) | Complex microservice meshes вҖ” let physics find the natural clusters |

### рҹ’Ҙ Blast Radius Simulation
Select any node and trigger a **failure simulation**. Watch as Rippler traces every downstream dependency in real time вҖ” pulsing Neon Crimson at the failure origin and cascading Warning Amber through every affected service, hop by hop. Know your blast radius before production does.

### рҹҺӣпёҸ Dependency Filters
Toggle edge types independently вҖ” show only Kafka flows, hide internal injections, focus on cross-service Feign calls. Cut through the noise.

### рҹ“Ұ Multi-Source Ingestion
Analyze projects from anywhere вҖ” no setup required on the target codebase.

### рҹ“· HD Architecture Export
Download the full graph as a high-resolution PNG. Drop it straight into your architecture docs, Confluence page, or README.

## рҹ§¬ What Rippler Detects

<br/>

### Nodes

```
рҹ”ө  SERVICE      в”Җв”Җ  Each microservice or application module
рҹҹў  CLASS        в”Җв”Җ  Java classes, interfaces, and Spring components  
рҹҹЈ  DATABASE     в”Җв”Җ  Detected from datasource config & @Entity mappings
рҹҹ   KAFKA_TOPIC  в”Җв”Җ  Message broker topics (producers & consumers)
в¬ң  PACKAGE      в”Җв”Җ  Package hierarchy for structural context
```

### Edges

```
в”Җв”Җв”Җв”Җв”Җв”Җ  FEIGN           OpenFeign client calls between services
в”Җв”Җв”Җв”Җв”Җв”Җ  REST            RestTemplate / WebClient HTTP calls  
в”Җв”Җв”Җв”Җв”Җв”Җ  KAFKA_PUBLISH   KafkaTemplate.send() вҖ” fire into the topic
в”Җв”Җв”Җв”Җв”Җв”Җ  KAFKA_CONSUME   @KafkaListener вҖ” pull from the topic
в”Җв”Җв”Җв”Җв”Җв”Җ  JPA_RELATION    Entity вҶ’ Database table mappings
в”Җв”Җв”Җв”Җв”Җв”Җ  INJECTION       Spring @Autowired / constructor injection
```

## рҹҺ® Usage Guide

<br/>

### Ingesting Your Project

**рҹ“Ғ Local Folder**
```
1. Click "Local / ZIP" tab
2. Enter the absolute path to your Spring Boot project root
3. Click "Ingest & Map"
```

**рҹ“Ұ ZIP Archive**
```
1. Click "Local / ZIP" tab
2. Drag & drop your project ZIP вҖ” or click "Select ZIP"
   вҡ пёҸ  Exclude target/ and node_modules/ to keep it small
3. Click "Ingest & Map"
```

**рҹҗҷ GitHub**
```
1. Click the "GitHub" tab
2. Paste a public repository URL
   e.g. https://github.com/user/my-spring-app
3. Click "Ingest & Map"
```

<br/>

### Exploring the Graph

| Action | How |
|---|---|
| **Switch View** | Toggle *Service Topology* вҶ” *Class Code Map* in the top bar |
| **Change Layout** | Switch *Semantic Flow* вҶ” *Force Orbit* in the toolbar |
| **Filter Edges** | Use *Dependency Filters* checkboxes in the sidebar |
| **Focus a Node** | Hover any node вҖ” unconnected nodes dim to reveal the chain |
| **Inspect a Node** | Click any node вҖ” detail panel slides in from the right |
| **Blast Radius** | Click *Simulate Failure* вҶ’ select a node вҶ’ watch it ripple |
| **Export PNG** | Click рҹ“· in the top-right toolbar |
| **Zoom Controls** | `+` `-` and fit-to-screen buttons in the toolbar |
